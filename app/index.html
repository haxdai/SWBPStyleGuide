| SWB Process - Guía de estilo

Introducción

La presente es una guía para la adaptación de la imagen gráfica de SWBProcess.

Uso

Bastará con agregar la clase swbp-body al contenedor donde se necesite aplicar los estilos de SWBProcess. Este contenedor puede ser el tag <body> o algún otro elemento de bloque. Para aplicarlo por ejemplo a todo el cuerpo del documento HTML:

<!DOCTYPE HTML>
<body class="swbp-body">
...
</body>

Botones

Tags para botones

Pueden usarse los tags para botones utilizados en bootstrap, es decir: <a> o <button>. La modificación particular para la imagen de SWBProcess consiste en hacer planos y cuadrados los botones.

Default Primary Success Info Warning Danger Link 
<a href="#" class="btn btn-default">Default</a>
<a href="#" class="btn btn-primary">Primary</a>
<a href="#" class="btn btn-success">Success</a>
<a href="#" class="btn btn-info">Info</a>
<a href="#" class="btn btn-warning">Warning</a>
<a href="#" class="btn btn-danger">Danger</a>
<a href="#" class="btn btn-link">Link</a>

Clases especiales para botones

Existen dos clases que particularmente diferencían los botones en SWBProcess de acuerdo a su utilidad en la UI. Estas son: btn-swbp-action y btn-swbp-action-head. La primer clase se utiliza en botones de acción principal dentro de los componentes de SWBProcess, mientras que la segunda se aplica en la barra superior de la interfaz y se ajusta en tamaño cuando se despliega como menú colapsado.

Action button Action header button 
<a href="#" class="btn btn-swbp-action">Action button</a>
<button class="btn btn-swbp-action">Action button</button>
<a href="#" class="btn btn-swbp-action-head">Action header button</a>
<button class="btn btn-swbp-action-head">Action header button</button>
